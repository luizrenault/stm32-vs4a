STVS4A Reference manual

 

 

This user manual describes the content of the STM32Cube software expansion package to enable Amazon® AVS® interfacing from an STM32application.

 

X-CUBE-VS4A package for STM32Cube provides application examples that connect STMicroelectronics boards to Amazon servers in order to ease implementation of AVS oriented products on STM32 devices.

X-CUBE-VS4A runs on the 32F769IDISCOVERY boards.

 

An implementation example shows how to implement a simple smart speaker.

X-CUBE-VS4A offers the following features:

 

·         Ready to run firmware example using Ethernet connectivity to support quick evaluation and development of AVS® device applications

·         Board configuration interface

·         Application specific Services

·         Connection to the Amazon® servers to manage token management and low layer communication

·         Provides an encapsulation of the protocol to ease application implementation.

 

General Information

 

This document describes the X-CUBE-VS4A package, the STVS4A middleware and the way to use it. It doesn’t explain Alexa® architecture or how to create an AVS account. Those descriptions and processes are available on the amazon website or on the developer website.

Security note

 

The application developer must take care of security aspects, and put in place mechanisms to protect the tokens and secrets, used to connect Alexa voice services. The application example provided in this package doesn’t do it. The example shows basic implementation, in order to ease the stack interface understanding.

 

STVS4A manages the connection between the ALEXA server and the device using secure connections. The AVS server policy will change after the delivery of this package. According to a message from amazon

 

Posted On: March 14, 2018

Alexa Developer:

The Alexa Voice Service is moving to Amazon Trust Services (ATS) certificates for all AVS endpoints. Because of this change, all Alexa-enabled devices and applications must support certificates vended from ATS by June 15, 2018. Your device or application will not be able to connect to AVS if it does not support ATS certificates after June 15, 2018.

We recommend testing that your device or mobile application successfully connects to AVS endpoints using ATS certificates. Please follow the instructions listed on the AVS Developer Forum to learn how to complete testing and upgrade support for ATS.

 

 

So, an action is required to change ATS certificate when STVS4A connects "api.amazon.com".  This need will be notified in the service serial console with the following message “TLS certification error, check the Root CA”.  The demonstration application already supports this change, but this support is neutralized since those Root CA are not supported yet by the server.

 

After June 15, 2018 you have to enable the new Amazon ATS certificates by uncommenting this line in service_asset.c

 

{"ding_notification_sound_wav", ding_notification_sound_wav, sizeof(ding_notification_sound_wav)},

{"beanforming_bmp", beanforming_bmp, sizeof(beanforming_bmp)},

{"Heart_bmp", Heart_bmp, sizeof(Heart_bmp)},

 

/* Your device or application will not be able to connect to AVS if it does not support ATS certificates after June 15, 2018. */

/*  Uncomment the following line to suport new ATS certificates */

{"default_amazon_root_ca", default_amazon_root_ca, sizeof(default_amazon_root_ca)},
