Release Notes for STVS4A Middleware

Copyright 2018 STMicroelectronics

 

Overview

 

STVS4A is a middleware designed to run on STM32 devices, in order to build Alexa Voice Service capable devices.

Please refer to Amazon® web site to discover AVS Cloud capabilities.

Contents

  • STVS4A offers an API to manage AVS exchanges with the cloud and with Audio front end / back end.
  • The communication protocol stack relies on LWIP ofr TCP/IP and MbedTls for TLS.
  • The example application uses the BSP (STM32 Audio BSP) for audio acquisition and rendering.
  • The AVS use cases are fully supported:
    • SpeechRecognizer / SpeechSynthetizer
    • Alerts
    • Speaker
  • The AudioPlayer use case is partially supported.
  • The stack has not been certified by Amazon.
  • STVS4A communicates with the cloud through secured http and http/2 flows. 
    • All other security aspects must be managed by the customer
  • The demo example shows the capabilities.
    • The board must be plug through ethernet to a direct internet access without security proxy.
    • The initiator product type can be changed through the touch screen
      • In Tap to talk mode,  long push on the blue button to initiate the communication with Alexa
      • In push to talk mode, push the blue button when talking

This application gives simple implementation example of the application services which the STVS4A framework calls.
